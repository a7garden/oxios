# Oxios v0.3.0 — Design Document

> oxi-sdk 0.14.0 adoption, credential management, onboarding flow, daemon mode, and CLI lifecycle.

---

## 0. Design Principles

| # | Principle | Implication |
|---|-----------|-------------|
| 1 | **Config-only** | Zero environment variable reads. Every setting lives in `~/.oxios/config.toml` or derived files. |
| 2 | **oxi-aware, not oxi-dependent** | If `~/.oxi/auth.json` (from `oxi setup`) exists → reuse it. If not → oxios manages its own credentials. No `oxi` CLI binary required. |
| 3 | **SDK as engine** | oxios depends on `oxi-sdk` (not raw `oxi-ai` / `oxi-agent`). The SDK's `OxiBuilder`, `KernelToolProvider`, `AgentGroup`, `MessageBus`, `AgentMetrics`, and `ProviderPool` become the kernel primitives. |
| 4 | **Daemon-first** | `oxios` runs as a background service. CLI is the control surface (`start`, `stop`, `status`, `restart`). |
| 5 | **Onboarding is mandatory** | First run without credentials → interactive wizard. After setup → daemon starts. |

## 1. Dependency Migration: `oxi-ai` + `oxi-agent` → `oxi-sdk`

### Current (v0.2.0)

```toml
# Cargo.toml
oxi-ai = { path = "../oxi/oxi-ai" }
oxi-agent = { path = "../oxi/oxi-agent" }
```

Kernel has its own `OxiEngineProvider` trait + wrapper that duplicates what `oxi-sdk::Oxi` already provides.

### Target (v0.3.0)

```toml
# Cargo.toml
oxi-sdk = { path = "../oxi/oxi-sdk" }
# oxi-ai and oxi-agent are transitive — NOT direct dependencies of oxios-kernel
```

#### What changes

| Module | Before | After |
|--------|--------|-------|
| `engine.rs` | Custom `EngineProvider` trait + `OxiEngineProvider` struct | Delete. Use `oxi_sdk::Oxi` directly. |
| `agent_runtime.rs` | Manual `AgentLoop::new()`, manual `ToolRegistry`, manual provider creation | `oxi_sdk::AgentBuilder` with `.kernel_tools()`, `.coding_tools()` |
| `agent_group.rs` | Custom implementation | Use `oxi_sdk::AgentGroup` with `GroupStrategy::Pipeline/Parallel/Orchestrated` |
| `event_bus.rs` | Custom `tokio::broadcast` channel | Keep for kernel-level events. Agent-to-agent comms use `oxi_sdk::MessageBus` |
| `metrics.rs` | Custom `AtomicU64` counters | Replace with `oxi_sdk::AgentMetrics` + `MetricsSnapshot` |
| `tools/` | Manual `ToolRegistry` + `AgentTool` impls | Implement `oxi_sdk::KernelToolProvider` trait to bridge kernel tools |

#### New `engine.rs`

```rust
use oxi_sdk::Oxi;

/// The kernel's engine — a thin wrapper around oxi-sdk's Oxi instance.
pub struct OxiosEngine {
    oxi: Oxi,
}

impl OxiosEngine {
    pub fn new(default_model: &str) -> Self {
        let oxi = oxi_sdk::OxiBuilder::new()
            .with_builtins()           // 50+ models, all providers
            .build();
        Self { oxi }
    }

    pub fn oxi(&self) -> &Oxi { &self.oxi }

    pub fn agent(&self, model_id: &str) -> oxi_sdk::AgentBuilder<'_> {
        let config = oxi_sdk::AgentConfig {
            model_id: model_id.into(),
            max_iterations: 20,
            ..Default::default()
        };
        self.oxi.agent(config)
    }
}
```

## 2. Credential Management

### Design

```
┌─────────────────────────────────────────────────┐
│  Credential Resolution (at request time)         │
│                                                   │
│  1. config.toml [engine].api_key ──┐             │
│  2. ~/.oxi/auth.json (if exists) ──┤ oxios reads │
│  3. oxi-ai env var fallback ───────┘ at startup  │
│                                                   │
│  At runtime: ProviderPool.resolve() → API key    │
└─────────────────────────────────────────────────┘
```

### Config structure

```toml
# ~/.oxios/config.toml

[engine]
default_model = "anthropic/claude-sonnet-4-20250514"
# Optional explicit API key — highest priority for oxios.
# If omitted, falls back to oxi auth store, then env vars.
api_key = ""

[engine.pool]
max_concurrency = 4
requests_per_minute = 60
```

### Credential priority (oxios perspective)

| Priority | Source | When |
|----------|--------|------|
| 1 | `config.toml` → `[engine].api_key` | Always checked first |
| 2 | `~/.oxi/auth.json` | Read if oxi CLI has been set up on this machine |
| 3 | `ProviderPool` rate-limited resolution | oxi-ai internal (env vars, OAuth, ambient) |

### In code

```rust
// crates/oxios-kernel/src/credential.rs

use anyhow::Result;

/// Credential store — reads from multiple sources with priority.
pub struct CredentialStore {
    /// Explicit API key from config.toml [engine].api_key
    config_key: Option<String>,
    /// Provider name to resolve (e.g. "anthropic")
    provider: String,
}

impl CredentialStore {
    pub fn new(provider: &str, config_key: Option<String>) -> Self {
        Self { config_key, provider: provider.into() }
    }

    /// Resolve the best available API key.
    pub fn resolve(&self) -> Option<String> {
        // 1. config.toml explicit key
        if let Some(ref key) = self.config_key {
            if !key.is_empty() { return Some(key.clone()); }
        }

        // 2. oxi auth store (~/.oxi/auth.json)
        if let Ok(Some(token)) = oxi_ai::oauth::load_token(&self.provider) {
            return Some(token.access_token);
        }

        // 3. oxi-ai env var fallback
        oxi_ai::get_env_api_key(&self.provider)
    }

    /// Store an API key to oxi's auth store.
    /// This is called by the onboarding wizard.
    pub fn store(provider: &str, api_key: &str) -> Result<()> {
        let token = oxi_ai::oauth::TokenBundle {
            access_token: api_key.into(),
            refresh_token: None,
            token_type: "Bearer".into(),
            obtained_at: chrono::Utc::now(),
            expires_in: 0, // 0 = never expires (raw API key)
            scope: None,
        };
        oxi_ai::oauth::save_token(provider, &token)?;
        Ok(())
    }

    /// Check if any credential is available.
    pub fn has_credential(&self) -> bool {
        self.resolve().is_some()
    }
}
```

### Security section simplification

```toml
# BEFORE (v0.2.0):
[security]
default_api_key = "sk-ant-..."    # ← removed
api_keys_path = "~/.oxios/api-keys.json"  # ← removed

# AFTER (v0.3.0):
[security]
auth_enabled = true
# API keys are managed via [engine].api_key or ~/.oxi/auth.json
```

The `security.default_api_key` and `security.api_keys_path` fields are **removed**. Credential management moves to `engine.api_key` + oxi auth store.

### Key logic

```rust
// crates/oxios-kernel/src/onboarding.rs

pub enum CredentialSource {
    /// Reused from existing ~/.oxi/auth.json
    OxiAuthStore { provider: String },
    /// Freshly entered by user → written to ~/.oxi/auth.json
    NewKey { provider: String, key: String },
    /// Written to config.toml [engine].api_key
    ConfigKey { key: String },
}

pub fn detect_credentials() -> Option<CredentialSource> {
    let provider = detect_provider_from_model(); // "anthropic" etc.

    // Check ~/.oxi/auth.json (oxi CLI credentials)
    if let Ok(Some(_)) = oxi_ai::oauth::load_token(&provider) {
        return Some(CredentialSource::OxiAuthStore { provider });
    }

    // Check config.toml [engine].api_key
    if let Some(key) = read_engine_api_key() {
        if !key.is_empty() {
            return Some(CredentialSource::ConfigKey { key });
        }
    }

    None // → run wizard
}

pub fn run_onboarding(oxios_home: &Path) -> Result<bool> {
    // 1. Check if already configured
    if let Some(source) = detect_credentials() {
        println!("Oxios credentials found via {:?}.", source);
        return Ok(false); // skip onboarding
    }

    // 2. Ask provider
    let provider = prompt_provider()?;

    // 3. Check if oxi auth.json has credentials for this provider
    if let Ok(Some(_)) = oxi_ai::oauth::load_token(&provider) {
        if prompt_bool("Use existing oxi credentials?") {
            // Nothing to store — already in auth.json
            return Ok(true);
        }
    }

    // 4. Ask for API key
    let api_key = prompt_api_key(&provider)?;

    // 5. Store in ~/.oxi/auth.json (shared with oxi CLI if installed)
    CredentialStore::store(&provider, &api_key)?;

    // 6. Write config.toml with model selection
    let model = prompt_model(&provider)?;
    write_engine_config(oxios_home, &model)?;

    Ok(true)
}
```

## 4. Daemon Mode

### Architecture

```
┌────────────────────────────────────────────┐
│  User                                      │
│    $ oxios start                           │
│    $ oxios status                          │
│    $ oxios stop                            │
│    $ oxios restart                         │
│    $ oxios daemon install                  │
└──────────┬─────────────────────────────────┘
           │
           ▼
┌────────────────────────────────────────────┐
│  PID file: ~/.oxios/oxios.pid              │
│  Socket:  ~/.oxios/oxios.sock              │
│  Logs:    ~/.oxios/logs/oxios.log          │
│  Config:  ~/.oxios/config.toml             │
└──────────┬─────────────────────────────────┘
           │
     ┌─────┴──────┐
     │  oxiosd    │  ← background process
     │  (daemon)  │
     └─────┬──────┘
           │
     ┌─────┴──────┐
     │  Gateway   │  → channels (web, cli, telegram)
     │  Kernel    │  → agents, tools, events
     │  Engine    │  → oxi-sdk Oxi instance
     └────────────┘
```

### CLI commands

```
$ oxios                     → same as `oxios start --foreground`
$ oxios start               → start daemon in background
$ oxios start --foreground  → start in foreground (for debugging)
$ oxios stop                → stop daemon via PID file
$ oxios restart             → stop + start
$ oxios status              → check if daemon is running, show PID, uptime, agent count
$ oxios log                 → tail the daemon log
$ oxios daemon install      → install launchd (macOS) / systemd (Linux) service
$ oxios daemon uninstall    → remove system service
```

### PID file management

```rust
// crates/oxios-kernel/src/daemon.rs

pub struct DaemonManager {
    pid_file: PathBuf,   // ~/.oxios/oxios.pid
    sock_file: PathBuf,  // ~/.oxios/oxios.sock (future: IPC)
    log_dir: PathBuf,    // ~/.oxios/logs/
}

impl DaemonManager {
    pub fn status(&self) -> DaemonStatus {
        // Read PID file, check if process is alive
        match self.read_pid() {
            Some(pid) if self.is_alive(pid) => DaemonStatus::Running { pid },
            Some(pid) => DaemonStatus::Stale { pid },
            None => DaemonStatus::Stopped,
        }
    }

    pub fn start(&self, config_path: &Path) -> Result<()> {
        match self.status() {
            DaemonStatus::Running { pid } => {
                bail!("oxios is already running (PID {})", pid);
            }
            DaemonStatus::Stale { .. } => {
                self.cleanup()?;
            }
            DaemonStatus::Stopped => {}
        }

        // Fork to background
        let child = std::process::Command::new(std::env::current_exe()?)
            .arg("start")
            .arg("--foreground")
            .arg("--config")
            .arg(config_path)
            .stdout(self.log_file()?)
            .stderr(self.log_file()?)
            .spawn()?;

        // Write PID
        self.write_pid(child.id())?;
        println!("oxios started (PID {})", child.id());
        Ok(())
    }

    pub fn stop(&self) -> Result<()> {
        match self.status() {
            DaemonStatus::Running { pid } => {
                // Send SIGTERM
                #[cfg(unix)]
                unsafe { libc::kill(pid as i32, libc::SIGTERM); }
                self.cleanup()?;
                println!("oxios stopped");
                Ok(())
            }
            DaemonStatus::Stale { .. } => {
                self.cleanup()?;
                println!("oxios cleaned up stale PID file");
                Ok(())
            }
            DaemonStatus::Stopped => {
                println!("oxios is not running");
                Ok(())
            }
        }
    }
}
```

## 6. oxi-sdk 0.14.0 Feature Adoption

### 6.1 KernelToolProvider

Replaces the manual tool registration in `agent_runtime.rs`.

```rust
// crates/oxios-kernel/src/tools/kernel_bridge.rs

pub struct OxiosKernelBridge {
    exec: Arc<ExecTool>,
    memory: Arc<MemoryTool>,
    browser: Option<Arc<BrowserTool>>,
    // ... other kernel tools
}

impl KernelToolProvider for OxiosKernelBridge {
    fn tool_names(&self) -> Vec<&str> {
        let mut names = vec!["exec", "memory", "program", "mcp",
                             "budget", "cron", "persona", "space"];
        if self.browser.is_some() { names.push("browser"); }
        names
    }

    fn register_tools(&self, registry: &ToolRegistry, ctx: &KernelToolContext) {
        registry.register_arc(Arc::clone(&self.exec) as Arc<dyn AgentTool>);
        registry.register_arc(Arc::clone(&self.memory) as Arc<dyn AgentTool>);
        if let Some(ref browser) = self.browser {
            registry.register_arc(Arc::clone(browser) as Arc<dyn AgentTool>);
        }
        // ...
    }
}
```

### 6.2 AgentGroup → Multi-agent Orchestration

Replace the custom `agent_group.rs` with `oxi_sdk::AgentGroup`.

```rust
use oxi_sdk::{AgentGroup, GroupStrategy};

// Pipeline: interview agent → planner agent → executor agent
let group = AgentGroup::new(GroupStrategy::Pipeline)
    .agent(interview_agent)
    .agent(planner_agent)
    .agent(executor_agent);

let result = group.run(prompt).await?;
```

### 6.3 MessageBus → Inter-agent Communication

```rust
use oxi_sdk::MessageBus;

// Created once, shared across all agents
let bus = MessageBus::new(256);

// Agent A publishes task completion
bus.publish(InterAgentMessage::broadcast(
    "agent-a", "task_complete", json!({"task": "research", "result": "..."})
));

// Agent B subscribes
let mut rx = bus.subscribe();
// In agent's event loop:
if let Ok(msg) = rx.try_recv() {
    if msg.is_for("agent-b") { /* handle */ }
}
```

### 6.4 AgentMetrics → Observability

```rust
use oxi_sdk::{AgentMetrics, MetricsSnapshot};

// Per-agent metrics
let metrics = AgentMetrics::new();

// After each run:
metrics.record_success(duration_ms, tokens_used, tool_calls);

// Dashboard API:
let snap = metrics.snapshot();
// snap.success_rate(), snap.avg_duration_ms(), snap.avg_tokens()
```

### 6.5 ProviderPool → Rate Limiting

```rust
use oxi_sdk::ProviderPool;

let pool = ProviderPool::new(provider, RateLimitPolicy {
    max_concurrency: config.engine.pool.max_concurrency,
    requests_per_minute: config.engine.pool.requests_per_minute,
});

// The pool wraps the provider and handles queuing + rate limiting
let agent = oxi.agent(config)
    .workspace(&workspace)
    .build()?;  // Pool is used internally by the provider
```

## 7. `main.rs` Rewrite

### Subcommand tree (v0.3.0)

```
oxios
├── start [--foreground]     # Start daemon (default: background)
├── stop                     # Stop daemon
├── restart                  # Restart daemon
├── status                   # Show daemon status + credentials + agents
├── log                      # Tail daemon log
├── run <prompt>             # One-shot prompt execution
├── chat                     # Interactive CLI session
├── config <show|get|set>    # Configuration management
├── pkg <install|list|...>   # Program management
├── agent <list|kill>        # Agent management
├── daemon <install|uninstall>  # System service management
├── program <name>           # Show program details
├── audit                    # Audit trail verification
├── git <log|tag>            # Git operations
├── budget [agent_id]        # Budget info
└── (no args)                # Same as `oxios start --foreground`
```

### Startup sequence

```
main()
 │
 ├─ 1. Parse CLI args
 ├─ 2. Load/create config.toml
 ├─ 3. Check credentials
 │     ├─ Has credentials? → continue
 │     └─ No credentials? → run_onboarding()
 │         └─ Onboarding completed? → print "run oxios start" → exit
 ├─ 4. Build OxiosEngine (oxi_sdk::OxiBuilder)
 ├─ 5. Build Kernel (supervisor, event bus, tools)
 ├─ 6. Dispatch subcommand
 │     ├─ start → DaemonManager::start() → fork to background
 │     ├─ start --foreground → cmd_serve() (current behavior)
 │     ├─ stop → DaemonManager::stop()
 │     ├─ status → DaemonManager::status() + KernelHandle::status()
 │     └─ ... (other subcommands)
 └─ 7. Shutdown on signal
```

## 10. Credential Flow Summary

```
User scenarios:

A) oxi CLI installed, user already ran `oxi setup`:
   $ oxios
   → Detects ~/.oxi/auth.json with "anthropic" token
   → "Credentials found via oxi. Starting..."
   → Daemon starts, agents use oxi's credential

B) Fresh machine, no oxi CLI:
   $ oxios
   → No credentials found
   → Onboarding wizard:
     → "Select provider: 1) Anthropic ..."
     → "Enter API key: sk-ant-..."
     → Stores in ~/.oxi/auth.json (shared format)
     → "Setup complete. Run 'oxios start'."
   $ oxios start
   → Daemon starts

C) oxi CLI installed later:
   → User runs `oxi setup` → ~/.oxi/auth.json updated
   → oxios picks up new credentials automatically (no restart needed)

D) Explicit config override:
   → User sets [engine].api_key in config.toml
   → This takes priority over auth.json
   → Useful for multi-tenant or per-project configs
```
