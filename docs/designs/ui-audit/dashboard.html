발견 사항 7건 · High 2 / Med 3 / Low 2

  • ①
    HIGH 승인 대기가 메트릭과 동등한 무게로 묻혀있음 — '승인대기'는 KPI 카드 6개 중 하나로 CPU%와 동일한 시각적 비중이고, 상세 큐는 페이지 맨 아래 조건부 렌더링. 대시보드에서 "지금 행동해야 할 것"이 수동 메트릭과 경쟁.routes/index.tsx:165-174 (카드) · :200 (큐)
  • ②
    HIGH 어포던스 모호 — 클릭 가능/불가 카드가 동일하게 보임 — CPU·RAM·Memory 카드는 href로 링크, AgentStatus는 Link, 토큰/분·승인대기는 비클릭. 세 가지 상호작용 패턴이지만 시각적으로 완전히 동일. 사용자가 무엇이 열리는지 발견 불가.stat-card.tsx:99-101 · agent-status-card.tsx:47 vs stat-card.tsx:173
  • ③
    MED KPI 6개 동일 비중 1행 — 그룹핑/강조 없음 — 중요도 순위나 그룹(에이전트 vs 시스템) 구분 없이 6등분. 스캔 리듬이 단조롭고 인지 부하가 높음.routes/index.tsx:116 (grid-cols-6)
  • ④
    MED Row 3이 이질적 기능을 동일 박스에 강제 — MCP / 예산 / Dream / Skills+Cron이 각기 다른 정보 밀도·액션을 갖는데 동일 카드에 우겨넣음. 시각 균질화가 기능 차이를 숨김.routes/index.tsx:192-197
  • ⑤
    MED 버전 배지 10px(text-2xs) + KPI 라벨 모두 대문자 tiny — 가독성 경계선, 반복적 타이포 리듬으로 위계 전달 약함.index.css:69 (--text-2xs:10px) · stat-card.tsx:106
  • ⑥
    LOW 승인 긴급도가 색상 단독 신호 — pending>0일 때 text-error로 표시. 텍스트 힌트는 있으나 주 신호가 색.routes/index.tsx:168-169
  • ⑦
    LOW 버전 배지 위치 — 타이틀과 분리되어 우상단 부유 — 메타는 타이틀 라인에 인라인으로 통합하면 위계가 명확해짐.routes/index.tsx:101-112

개선안 — 무엇이 바뀌는가

  • 액션 센터 승격: 승인 대기가 있으면 KPI 위에 풀폭 알림 스트립. 없으면 조용한 '이상 없음' 상태. 행동 항목이 메트릭과 경쟁하지 않음.
  • 어포던스 통일: 모든 KPI 카드를 균일하게 클릭 가능하게(↗ 큐브 표시)하거나, 읽기 전용 메트릭(RAM)은 시각적으로 평평하게 차등화. ② 해결.
  • KPI 그룹핑: 6등분 → 「에이전트·메모리」「시스템 바이탄」 두 클러스터 + 서브헤더. 위계와 스캔 속도 개선. ③ 해결.
  • Row 3 재배치: 이질적 4박스 제거. 무게가 큰 것(예산)은 2열로, 자동화(Dream/Cron/Skills)는 테마별 그룹으로. ④ 해결.
  • 메타 인라인 통합: 버전 배지를 타이틀 서브라인으로. ⑤⑦ 해소.
