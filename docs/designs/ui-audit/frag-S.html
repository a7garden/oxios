발견 사항 — 7건 (1 HIGH · 3 MED · 3 LOW)

  • ①
    HIGH 세션 삭제에 확인·되돌림 없이 즉시 API 호출 — Trash2 아이콘 버튼 클릭 한 번으로 deleteMutation.mutate(row.id). 대화 기록이 영구 삭제되며 확인 모달·Undo 토스트 부재. Personas·Cron과 동일한 무방비 파괴 패턴(전액 패턴 ③).routes/sessions/index.tsx:80-83 (deleteMutation.mutate) · :29-32
  • ②
    MED "Agent" 컬럼이 실제로는 user_id를 렌더 — 헤더는 t('sessions.agent')(에이전트)지만 accessor는 row.user_id.slice(0,8)…. 컬럼의 보이는 라벨과 실제 데이터가 불일치하여 사용자가 "이 세션의 에이전트가 누구인지"가 아니라 사용자 ID를 보고 있게 됨.routes/sessions/index.tsx:53-55 (header vs accessor)
  • ③
    MED refetchInterval 10s가 탭 가시성과 무관하게 백그라운드 폴링 — refetchInterval: 10000이 document visibility와 무관. 백그라운드 탭에서 의미 없는 fetch가 계속됨 (Resources·Cron과 동일 패턴).routes/sessions/index.tsx:26
  • ④
    MED 행 전체 클릭(상세 이동) + 행 안 delete 버튼이 동일 평면 — DataTable onRowClick이 상세로 이동시키고, delete 버튼은 stopPropagation으로 분리. 행이 클릭 가능하다는 시각 단서(hover 배경·chevron)가 없어 "어디를 누르면 상세로 가는지" 발견 불가, 삭제와 이동이 혼재.routes/sessions/index.tsx:76-90 (delete) · :112-119 (onRowClick)
  • ⑤
    LOW ID 컬럼이 12자 truncate, 복사·풀ID 툴팁 없음 — row.id.slice(0,12)…로 잘린 mono 텍스트만. 전체 ID 확인·복사 불가(상세 페이지에서도 $sessionId.tsx:106에 mono 텍스트만).routes/sessions/index.tsx:43-45
  • ⑥
    LOW 7열 중 4개가 mobile hidden — 모바일에서 세션 삭제 불가 — ID·생성·갱신·삭제(delete) 컬럼 모두 mobilePriority:'hidden'. 모바일 사용자는 상세 진입만 가능하고 리스트에서 삭제할 수 없음.routes/sessions/index.tsx:42 · :64 · :69 · :75
  • ⑦
    LOW 총 건수만 표시 — 활성/종료 분류·필터·검색·정렬 없음 — t('sessions.registered', {`{count}`})로 총합만. 활성 세션이 얼마인지, 어느 세션이 최근인지 한눈에 안 들어오고 테이블 자체에 search/sort 없음.routes/sessions/index.tsx:99 · :112-119

개선안 — 무엇이 바뀌는가

  • 삭제 → shadcn AlertDialog 확인 + Undo 토스트: 즉시 mutate 대신 확인 다이얼로그 + 5초 되돌리기. 전액 파괴 액션 가이드라인 적용. ① 해결.
  • "Agent" 컬럼 정정: 세션에 실제 agent 필드를 매핑하거나, 의도가 사용자 식별이면 헤더를 "User"로 변경. 라벨=데이터 정합성. ② 해결.
  • refetch 가시성 가드: refetchInterval: (q) => document.hidden ? false : 10000. Resources·Cron에도 동일 적용 권장. ③ 해결.
  • 행 어포던스 + 삭제 분리: 행 hover 배경·우측 chevron으로 "클릭=상세" 신호, delete는 별도 고정 셀(모바일은 ⋯ 메뉴). ④·⑥ 해결.
  • ID 복사 버튼 + 풀ID 툴팁: truncate ID에 click-to-copy. ⑤ 해결.
  • 검색/필터(활성·종료)/정렬 툴바 + 활성 카운트 배지: 헤더에 활성/종료 분류와 검색 인풋. ⑦ 해결 · orphan route는 Sessions를 콘솔 nav에 승격(또는 Chat 그룹)하여 활성 상태 부여.
