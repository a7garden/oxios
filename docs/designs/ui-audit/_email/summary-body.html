— 재발 패턴 Top 3 —
SYSTEMIC ① 액션-메트릭 위계 붕괴
행동이 필요한 항목(승인대기·실패·실행중 세션·임계치 초과)이 수동 메트릭과 동일 시각 무게로 렌더된다. "지금 뭘 해야 하나"가 한눈에 안 보인다.

Fix: 전 앱 "액션 센터" 패턴 — 행동 항목을 상단 풀폭 알림/우선 영역으로 승격, 메트릭과 분리.
재발: Dashboard(index.tsx:165) · Agents(agents/index.tsx:69) · Token Maxing(status-header.tsx:11) · Resources(resources.tsx:68)
SYSTEMIC ② 발견성 부채 — hover-only
주요 액션이 hover에서만 나타나거나 주 진입점이 부유/숨김 상태. 키보드·터치·스크린리더 사용자가 액션을 발견 못 한다.

Fix: 액션 항상 노출 + focus-visible 보장, 빈 상태에 primary CTA 인라인, Settings를 메인 nav로 승격.
재발: Projects(project-card.tsx:40) · Mounts(mounts/index.tsx:102) · MCP(mcp.tsx:28) · Settings(sidebar-footer.tsx:71)
SYSTEMIC · 신규 ③ SDK role / 라우팅 묵음
엔진·설정에 멀티 프로바이더/role 라우팅이 구현되어 있지만 chat·dashboard가 그 결정에 무지. 사용자는 "지금 어떤 모델로 가고, 왜 그 모델인지, 쿼터는 얼마나 남았는지"를 전혀 알 수 없다. 게다가 role 셀렉터 자체가 컴파일 에러로 비어 있다. 결과: 라우팅이 활성인데 UI가 끊겨있어 "내가 어떤 모델을 쓰는지 모름 → 다른 모델로 못 바꿈 → 토큰 없으면 멈춤".

Fix: ① chat-input.tsx:66의 const { roles = [] } = props destructure 한 줄 수정 (TypeScript 컴파일 에러 TS2304: Cannot find name 'props' 현재 발생 중) ② role pill + 쿼터 뱃지 + 라우팅 사유 표면 ③ idle timeout + provider-quota 조회 기반 인라인 에러 카드.
재발: Chat(chat-input.tsx:66, chat.ts:1289-1315) · Dashboard SystemHealthCard(providers-section.tsx)
