발견 (Findings)

  • HIGH

    카드 액션 버튼이 아이콘 전용·32px 미만·반투명 기본값으로 키보드/저시력 사용자에게 노출되지 않음

    편집/삭제 버튼이 p-1(≈24×24px) + opacity-70 group-hover:opacity-100로 감춰져, WCAG 2.5.5(타겟 ≥44×44px CSPC)·1.4.13(콘텐츠 간지지) 미달. 키보드 사용자는 focus-within에만 희미하게 보이며 SR 사용자는 title="Edit"만으로 의도를 파악해야 한다. 시각 사용자도 호버 전에는 “정적 카드”로 오인한다.

    components/project/project-card.tsx:40-59 (opacity-70, p-1, h-3.5 w-3.5 icon)

  • MED

    검색 인풋 옆에 결과 카운트가 조건부로만 노출

    search가 비어 있으면 카운트가 DOM에서 사라진다({search && (...)}). 검색 결과 0건·오타 상황에서도 화면이 비어 보여 “왜 결과가 없는지” 시그널이 없다. 빈 상태(EmptyState)로 가야만 메시지가 보인다.

    routes/projects/index.tsx:58-62 ({`{search && ...}`}) · :70-91 EmptyState 분기

  • MED

    source 배지가 i18n 키 없이 원문 노출

    카드는 SOURCE_COLORS로 색만 매핑하고 라벨은 project.source ?? 'manual'을 그대로 표시. 영문 값(manual / auto_detected)이 그대로 노출되어 UI의 다른 한국어 라벨과 어긋난다.

    components/project/project-card.tsx:14-17, 36-38 (string 그대로)

  • LOW

    헤더의 "New Project"가 시스템 기본 라벨 영어 그대로

    projects.title/desc/new가 한국어 key 없이 'Projects' / 'New Project' 기본값을 그대로 가지고 있어 page-h1/sub가 영문으로 노출된다. 하위 카드/검색 placeholder도 동일하게 'No projects found' 등 영문 기본값.

    routes/projects/index.tsx:36-38, 53, 75-81 (기본값)

  • LOW

    paths[0]만 표시해 다중 경로 프로젝트의 맥락이 사라짐

    프로젝트는 paths: string[]이지만 카드에는 첫 경로만 line-clamp-1 truncate으로 노출. 프로젝트 상세(/projects/$projectId)로 이동하기 전에는 “몇 개의 경로/어느 마운트에 연결돼 있는지” 알 수 없다.

    components/project/project-card.tsx:68-72 (paths[0]) · types: Project.paths

변경 제안

  • 카드 액션을 h-8 w-8 이상으로 키우고 라벨 동반(편집 · 삭제), opacity-70 group-hover 제거하고 항상 노출. 파괴 액션은 hover 시 confirm 모션(예: 라벨이 “삭제하시겠어요?”로 1회 변경) 부여.
  • useTranslation로 manual/auto_detected/paths plural 추가. 기본 라벨을 한글로.
  • 검색 인풋 옆에 카운트(0 포함)와 “결과 없음” 인라인 메시지 상시 표시. EmptyState의 액션은 그대로.
  • paths.length > 1일 때 “+N paths” 칩과 “연결된 마운트 N개” 힌트 노출.

발견 (Findings)

  • HIGH

    카드 액션이 아이콘 전용·32×32px·opacity-70 기본값으로 시각 사용자도 호버 전에 인식 불가

    편집/갱신/삭제 버튼이 h-8 w-8(32px, WCAG ≥44px 미달) + opacity-70 group-hover:opacity-100 패턴으로 항상 어둡게 노출. “편집/갱신/삭제”를 가르는 신호가 아이콘 색 뿐이라, 시각 사용자는 호버하거나 focus-within하기 전엔 카드를 “정적 콘텐츠”로 오인한다. 동형(同形) 아이콘(✎/↻)은 색맹 사용자에게 구분 불가.

    routes/mounts/index.tsx:113-141 (absolute, opacity-70, h-8 w-8)

  • MED

    “갱신 필요” 배지의 트리거 액션이 카드 위에 명확히 표시되지 않음

    enrichment_pending 배지(bg-amber-500/10)는 상태만 알리고, 이를 해소할 1차 액션(“즉시 갱신”)이 일반 카드의 갱신 버튼과 모양이 같아 사용자가 차이를 알기 어렵다. 보색 배지가 같은 호버 시그널 안에 묻혀 사실상 “눈에 띄는” 우선순위를 잃는다.

    routes/mounts/index.tsx:156-160 (enrichment_pending 배지) · :123-131 (갱신 버튼)

  • MED

    새로고침 RefreshButton이 검색 인풋 옆이지만 “갱신 필요” 마운트 일괄 재시작과 무관

    RefreshButton은 refetch()만 호출(index.tsx:81)하고 카드의 rescanMount(:50)는 아니다. “갱신 필요 N” 칩이 있어도 일괄로 해결할 수 없어 작업이 분절된다.

    routes/mounts/index.tsx:81 (RefreshButton) · :35 (rescanMount) · 메모리 overview의 EnrichNow과 비대칭

  • LOW

    헤더 타이틀/CTA가 영문 기본값으로 노출

    mounts.title/desc/create/empty/emptyDesc/rescan/rescanned이 영문 폴백을 그대로 가지고 있어 한국어 로케일에서도 “Mounts / New Mount”가 그대로 보인다. 다른 페이지가 한국어인 상태에서 한 화면만 어긋난다.

    routes/mounts/index.tsx:62-69, 91-100 (t() 기본값)

  • LOW

    카드 그리드가 sm:grid-cols-2 lg:grid-cols-3로 3열 고정, ≥2xl에서 더 늘어나지 않음

    콘텐츠 영역이 1200px을 넘어가도 카드는 3열로 고정되어 우측에 큰 여백이 생긴다. 같은 .g3을 쓰는 프로젝트/페르소나 화면과 함께 “콘텐츠가 끝났다”는 신호가 약하다.

    routes/mounts/index.tsx:105 (grid gap-3 sm:grid-cols-2 lg:grid-cols-3)

변경 제안

  • 카드 액션을 h-9 이상 pill 버튼으로 항상 노출(✎ 편집, ↻ 갱신, 🗑 삭제), group-hover opacity 제거. 파괴 액션은 1차 confirm.
  • enrichment_pending 마운트만 갱신 버튼을 warn 톤으로 강조 + “즉시 갱신” 카피. 필터 칩과 연동하여 “갱신 필요 · 1” 선택 시 일괄 작업 노출.
  • RefreshButton 옆에 “enrichment_pending 마운트 일괄 재시작” 추가 액션 — 마운트 페이지의 자기완결성↑.
  • 한글 i18n 키 보강, 그리드를 lg:grid-cols-3 2xl:grid-cols-4로 확장.

    components/memory/memory-overview.tsx:23-49 (DistBar) · :65-66 (max 산출)
