발견 사항 5건 · High 2 / Med 2 / Low 1

  • ①
    HIGH 잡 삭제에 확인·되돌림 없이 즉시 API 호출 — Trash2 아이콘 클릭만으로 deleteMutation.mutate(job.id). 예약 작업이 영구 삭제되며, 운영 중인 백업/동기화가 조용히 멈춤. (Personas와 동일한 파괴적 액션 패턴)routes/cron-jobs.tsx:161-168
  • ②
    HIGH cron schedule 입력이 자유 텍스트 — 검증·미리보기·문법 도움말 없음 — Input placeholder만 있고 실제 cron 신택스 검증도, "다음 3회 실행" 미리보기도 없음. 잘못된 스케줄이 조용히 등록됨.routes/cron-jobs.tsx:90-94
  • ③
    MED toggle/delete가 ghost icon-only 버튼 — 파괴(delete)와 일반(toggle)이 시각적으로 동등 — 두 버튼이 같은 ghost 스타일, delete만 색으로 구분. 아이콘 인식에 의존.routes/cron-jobs.tsx:152-169
  • ④
    MED refetchInterval 10초가 탭 가시성과 무관하게 폴링 — 백그라운드 탭에서 의미 없는 fetch 지속 (Resources 화면과 동일 패턴).routes/cron-jobs.tsx:33
  • ⑤
    LOW 잡에 마지막 실행 결과(성공/실패) 표시 없음 — last_run 타임스탬프만 있고 성공 여부 미상. 실패한 잡을 눈치채기 어려움.routes/cron-jobs.tsx:139-150

개선안 — 무엇이 바뀌는가

  • 삭제 확인 다이얼로그: icon → "삭제…" 라벨 버튼 + 확인 모달. ① 해결 (전 앱 파괴적 액션 가이드라인 적용).
  • cron 검증 + 미리보기: schedule 입력 하단에 문법 도움말 + "다음 실행: 03:00, 03:15, 03:30" 미리보기. ② 해결.
  • 액션 라벨화: toggle/delete를 icon-only에서 라벨 버튼으로, destructive는 경계 색. ③ 해결.
  • 가시성 인식 폴링 + 결과 배지: document.hidden 시 폴링 중지, 잡 카드에 ✓/✗ 결과 배지. ④⑤ 해결.
