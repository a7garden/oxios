전신적 일관성 패턴 — 상위 8종

개별 화면을 넘어 2개 이상의 탭에서 반복되는 분열의 뿌리. 이 패턴들이 일관성 점수를 갉아먹는 주원인이며, 가장 높은 수정 우선순위를 갖는다. 각 패턴은 발생 지점(file:line)과 한 줄 해결 방향을 포함한다.

P1 · HIGH

페이지 헤더 파편화 (전 탭 반복)

같은 역할의 헤더가 탭마다 5가지 축(제목 굵기·정렬·부제 크기·앞아이콘·액션 갭)에서 분화. 가장 큰 시각적 분열원. <PageHeader> 공유 컴포넌트 하나로 전부 흡수 가능.

제목 굵기: Dashboard만 font-semibold tracking-tight routes/index.tsx:107 vs 전 탭 font-bold
헤더 정렬: Dashboard만 items-end routes/index.tsx:105
부제 크기: 10면 text-muted-foreground(크기 미지정) vs 4면 text-sm — mcp/memory/cron/email/git/personas/resources/security/skills/workspace vs budget/token-maxing/mounts/projects
제목 앞 아이콘: skills/agents/sessions만 부분 적용 (routes/skills.tsx:269, routes/agents/index.tsx:320, routes/sessions/$sessionId.tsx:101)
액션 클러스터 갭: gap-2 vs gap-3 vs flex gap-2
P2 · HIGH

표면 간격 리듬 이중화

페이지 루트 간격이 space-y-6과 space-y-4로 갈라짐. 의도적 밀도 구분의 근거 없음 → 시각적 리듬이 탭마다 흔들린다.

space-y-6 (12면): budget·cron·email·git·mcp·memory·personas·resources·security·skills·sessions·agents-detail
space-y-4 (5면): routes/index.tsx:103 · routes/agents/index.tsx:316 · routes/mounts/index.tsx:57 · routes/projects/index.tsx:31 · routes/projects/$projectId.tsx:207
→ space-y-6 단일화 권장
P3 · MED

버튼 아이콘 간격 — margin vs flex gap

생성 버튼의 Plus 아이콘을 mr-1(margin)로 띄우는 경우와 flex gap으로 띄우는 경우가 혼재. Button이 gap를 내장하므로 margin는 불필요.

routes/mcp.tsx:36 · routes/personas.tsx:122 (mr-1) vs Button 내장 gap 기반 페이지들
P4 · HIGH

사이드바 내비게이션 아이콘 — 중복 + 의미 오정합

탐색 아이콘이 기능과 매치되지 않거나 중복되어 웨이파인딩을 해친다. 단일 행 변경으로 즉각 수정 가능한 대비 효과.

중복: Zap가 Skills(sidebar.tsx:97) + MCP(sidebar.tsx:137) 양쪽 — 같은 아이콘 = 다른 기능
의미 오정합: Security = Bell(sidebar.tsx:146) — Bell은 알림 관례, Security는 ShieldCheck가 정합. 알림은 Header의 MenuClock→NotificationCenter가 이미 담당.
P5 · MED

탭 패턴 — 트리거 아이콘·콘텐츠 래핑 불일치

다중 탭 페이지(MCP·Memory·Budget·Skills)가 트리거 아이콘 유무와 콘텐츠 Card 래핑 유무를 각자 다르게 결정. "모두 또는 아무것도" 원칙 위반.

트리거 아이콘: MCP(routes/mcp.tsx:51-59)만 있음; Memory/Budget/Skills 무아이콘
콘텐츠 Card 래핑: MCP는 <Card><CardContent className="pt-6">로 래핑(routes/mcp.tsx:62-84); Memory는 비래핑
P6 · LOW

입장 애니메이션 부분 적용

animate-fade-in-up가 일부 페이지에만. 전 탭 또는 무적용으로 통일.

적용: routes/index.tsx:103 · routes/agents/index.tsx:316 · 미적용: mcp·memory·budget·skills 등
P7 · MED

Settings 패러다임 단절

/settings가 3-zone SettingsShell(레일+본문+리뷰)로 헤더·간격·탭 패턴이 전 탭과 단절. 복잡 설정에는 적절하나 경계 명시 없이 다른 탭과 나란히 놓여 일관성으로 인지됨.

components/settings/settings-shell.tsx:65 (SettingsShell) · routes/settings.tsx:838 (renderActiveSection)
P8 · LOW

Persona 헤더 3줄 (고유 예외)

유일하게 헤더가 title+subtitle+hint 3줄. 고유하지만 비일관 — hint는 부제에 통합 또는 본문으로 이동.

routes/personas.tsx:115-117 (singleActiveHint, text-xs 3째 줄)
종합. 전 탭에 공통으로 반복되는 분열(P1·P2·P3)은 PageHeader 공유 컴포넌트 + 간격·버튼 규칙 단일화 한 방으로 대부분 흡수된다. 사이드바 아이콘 정정(P4)은 두 줄 변경이지만 발견성 향상이 즉각적이므로 최우선. 아래 "개선 설계" 사양 섹션에 통일 시스템을, 각 화면 섹션에 현재↔제안 비교를 제시한다.
