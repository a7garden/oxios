발견 (Findings)

  • MED

    섹션 이동이 브라우저 히스토리에 push되지 않는다

    handleNavigate가 window.history.replaceState로 URL만 바꾼다. 그래서 브라우저 뒤로가기로 직전 섹션으로 돌아갈 수 없다. 설정 섹션 탐색 히스토리가 존재하지 않는다.

    routes/settings.tsx:738-744 (handleNavigate, replaceState 사용)

  • MED

    재시작 필요 변경을 저장(diff 미리보기) 전에 미리 알기 어렵다

    어떤 필드가 핫리로드되고 어떤 것이 재시작을 요구하는지는 configMeta로 분류되지만, 사용자에게는 DiffPreview 모달을 열어야만 보인다. 섹션 내에서 인라인으로 미리 알 수 없어 "저장하면 재시작될지" 예측이 어렵다.

    routes/settings.tsx:622-665 (hotReload/restartScope 분류) · :813-820 (DiffPreview에서만 노출)

  • MED

    설정 진입점이 사이드바 푸터 톱니바퀴에만 있다

    설정은 콘솔 네비게이션 그룹에 항목으로 없고, 사이드바 하단의 푸터 아이콘으로만 진입한다. 발견성이 작은 아이콘에 의존하며, 다른 주요 화면과 내비게이션 일관성이 떨어진다.

    routes/settings.tsx:778-796 (SettingsShell) · components/layout/sidebar.tsx:229-231 (footer 진입)

  • LOW

    SaveDock이 position:fixed로 떠 있어 하단 콘텐츠를 가린다

    미저장 변경이 있을 때 SaveDock이 화면 하단에 고정된다. 긴 설정 폼의 마지막 필드를 가릴 수 있어 저장 전 검토가 방해받는다.

    routes/settings.tsx:798-810 (SaveDock position:fixed)

  • LOW

    폐기가 전체(discard all)만 가능 — 섹션/필드별 폐기 불가

    handleDiscard는 전체 폼을 서버 상태로 되돌린다. 특정 섹션만, 혹은 특정 필드만 선택적 폐기가 불가하여 실험적 변경을 부분 취소하기 어렵다.

    routes/settings.tsx:719-726 (handleDiscard 전체 리셋)

변경 제안

  • 섹션 이동을 pushState(또는 라우터 search)로 변경 — 뒤로가기 지원.
  • 필드 라벨 옆에 인라인 재시작/즉시적용 태그 표시 — diff 전 예측 가능.
  • 설정을 사이드바 시스템 그룹에 정식 항목으로 추가 (발견성).
  • SaveDock에 의해 가려지지 않도록 폼 하단 패딩 확보.
