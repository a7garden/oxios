발견 (Findings)

  • MED

    뷰 간 기능 비대칭 — 상태 탭 필터가 테이블 뷰에만 존재

    전체/실행/완료/실패 상태 탭과 카운트 뱃지는 테이블 뷰에서만 렌더된다. 캔버스(기본 뷰)에서는 상태별 필터링이 불가하여, 그래프에 떠 있는 노드를 상태로 좁힐 방법이 없다.

    routes/agents/index.tsx:421-459 (Tabs, table-view 블록 내) · :380-391 (캔버스엔 필터 없음)

  • MED

    통계 표시가 뷰마다 두 가지 형태로 갈라진다

    테이블 뷰는 StatsBar 컴포넌트로, 캔버스 뷰는 인라인 텍스트(3 running · $cost · tokens)로 통계를 보여준다. 동일 정보가 두 형태로 제공되어 일관성이 떨어진다.

    routes/agents/index.tsx:59-78 (StatsBar) · :318-329 (캔버스 인라인 통계)

  • MED

    에이전트 시작/포크 액션이 화면에 없다 (모니터링 전용)

    대시보드와 마찬가지로 에이전트 목록 화면에도 "새 에이전트 시작" CTA가 없다. Agent OS의 핵심 액션이 사이드바/헤더로만 접근된다. 빈 캔버스는 특히 행동 유도가 부재하다.

    routes/agents/index.tsx:307-516 (CTA 없음)

  • LOW

    페이지네이션 버튼이 영문 하드코딩

    "← Previous" / "Next →" 가 t() 없이 문자열 리터럴로 박혀 있다. 한국어 UI에서도 영어로 표시된다.

    routes/agents/index.tsx:495, 503

  • LOW

    캔버스 기본값이 첫 진입 시 정보 부족

    기본 뷰는 캔버스이지만 실행 중인 에이전트가 없으면 빈 그래프만 보인다. "무엇을 할 수 있는가"에 대한 유도(최근 에이전트, 시작 버튼)가 없다.

    routes/agents/index.tsx:194 (view ?? 'canvas') · :380-391

변경 제안

  • 상태 탭/필터를 캔버스 뷰에도 노출 (또는 뷰 전환 시 필터 상태 공유).
  • 통계 표시를 단일 StatsBar로 양 뷰 통일.
  • 헤더에 "새 에이전트" CTA 추가; 빈 캔버스에 시작 유도 + 최근 에이전트 목록.
  • 페이지네이션 문자열 i18n 적용.

발견 (Findings)

  • MED

    활성 페르소나를 비활성화할 인-카드 방법이 없다 (단방향 토글)

    활성화 버튼은 {!persona.enabled && ...} 조건으로 활성 페르소나에서 사라진다. 한 번 활성화하면 카드에서 끌 수 없고, 다른 페르소나를 활성화해야만 교체된다. 토글이 단방향이다.

    routes/personas.tsx:228-237 (활성화 버튼, enabled일 때 숨김)

  • MED

    "활성" 상태가 별 아이콘(filled, 경고색)에만 의존

    활성 상태가 색이 채워진 작은 별 하나로만 표시된다. "활성"이라는 텍스트 라벨이 없어 색맹·초기 사용자가 상태를 즉시 파악하기 어렵다.

    routes/personas.tsx:213 (Star className="text-warning fill-warning")

  • LOW

    단일 활성 vs 다중 활성 규칙이 UI에 설명 없음

    여러 페르소나를 동시에 켤 수 있는지, 한 번에 하나인지가 UI 어디에도 표시되지 않는다. 활성화 동작의 결과를 예측하기 어렵다.

    routes/personas.tsx:228-237 (활성화 로직에 안내 없음)

  • LOW

    카드에 시스템 프롬프트 미리보기가 없다

    카드는 role만 표시하고 실제 시스템 프롬프트 내용은 보여주지 않는다. 프롬프트를 확인하려면 편집 다이얼로그를 열어야 한다.

    routes/personas.tsx:248-253 (role만 렌더)

  • LOW

    생성 다이얼로그의 설명 입력이 한 줄 Input

    설명 필드가 단일 줄 Input이다. 페르소나 설명이 길어지면 입력이 잘리고 시스템 프롬프트처럼 Textarea 여백이 부족하다.

    routes/personas.tsx:155-162 (description Input) vs :164-175 (systemPrompt Textarea)

변경 제안

  • 활성 페르소나 카드에 "활성 끄기" 토글 추가 — 단방향을 양방향으로.
  • 별 아이콘에 "활성" 텍스트 뱃지 병기 — 색 의존 제거.
  • "한 번에 하나만 활성화됩니다" 안내 문구 추가.
  • 카드에 시스템 프롬프트 2줄 미리보기 추가.

발견 (Findings)

  • HIGH

    탭(설치됨/마켓플레이스) 상태가 URL에 동기화되지 않는다

    초기 진입 시에만 search.tab을 읽지만, 이후 탭 전환(handleTabChange)은 setTab만 호출하고 URL을 바꾸지 않는다. 새로고침·공유·브라우저 뒤로가기 시 현재 탭이 유실된다. /marketplace 리다이렉트가 ?tab=marketplace로 보내지만 그 이후엔 URL이 의미를 잃는다.

    routes/skills.tsx:121 (초기값만 search.tab) · :247-252 (handleTabChange, URL 미갱신) · marketplace.tsx:4 (리다이렉트)

  • MED

    마켓플레이스 두 소스(clawhub / skills.sh)의 빈 상태가 비대칭

    검색어가 비어 있을 때 skills.sh는 트렌딩 목록으로 폴백하지만, clawhub는 deferredQuery.trim().length > 0 조건으로 쿼리 자체가 비활성화되어 빈 화면만 보인다. 같은 탭의 두 소스가 전혀 다른 첫 화면을 제공한다.

    routes/skills.tsx:185 (clawhub enabled 조건) · :359 (skills.sh trending 폴백)

  • MED

    요약 pill이 수동 새로고침을 대체하여 사용자 제어가 없다

    이전의 새로고침 버튼이 SkillSummaryPill(필터 토글)로 대체되었다. 30초 자동 갱신에만 의존하므로, 설치 직후나 외부 변경 시 즉시 갱신할 수단이 없다.

    routes/skills.tsx:170 (refetchInterval 30000) · :268 (SkillSummaryPill이 새로고침 대체)

  • LOW

    /marketplace가 실제 화면이 아니라 리다이렉트

    /marketplace는 <Navigate to="/skills" search={{tab:'marketplace'}}>일 뿐 별도 화면이 아니다. 별도 라우트처럼 보이지만 실제는 스킬 페이지의 두 번째 탭이어서 내비게이션에 혼란을 준다.

    routes/marketplace.tsx:3-5

  • LOW

    업데이트 가능 표시가 마켓플레이스 탭 뱃지에만

    UpdateBadge가 마켓플레이스 탭에만 표시된다. 설치됨 탭에서는 어떤 스킬에 업데이트가 있는지 한눈에 알기 어렵다(개별 카드를 봐야 함).

    routes/skills.tsx:326 (UpdateBadge on marketplace tab) · :312-313 (installed 탭엔 카운트만)

변경 제안

  • 탭 전환 시 navigate({search:{tab}})로 URL 동기화 — 새로고침·공유·히스토리 유지.
  • clawhub 빈 검색 시 추천/인기 폴백 추가 — 두 소스의 첫 화면 통일.
  • 요약 pill 옆에 수동 새로고침 버튼 복원.
  • 설치됨 탭에도 업데이트 가능 스킬 수 표시.
