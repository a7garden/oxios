개선 설계 — 통일 디자인 사양

PageHeader · 간격 · 버튼 · 아이콘맵 · 탭 · 상태

각 탭의 "제안" 패널이 수렴하는 정규형(canonical)을 단일 사양으로 고정한다. 이 사양은 기존 다수 패턴을 근거로 삼되, 분열된 지점(헤더·간격·버튼 아이콘)은 하나로 좁힌다. 구현 시 공유 컴포넌트(<PageHeader>)로 캡슐화해 분산 변형을 원천 차단한다.

1. PageHeader (공유 컴포넌트화)

모든 Console 탭의 헤더를 단일 컴포넌트로. 제목·부제·우측 액션 슬롯. Dashboard의 font-semibold tracking-tight·items-end 변형을 제거하고 전 탭 동일 적용.

정규 PageHeader
페이지 제목
text-sm text-muted-foreground — 항상 명시
<div class="space-y-6">
  <PageHeader title=… subtitle=…>
    <RefreshButton/> <Button><Plus/> 생성</Button>
  </PageHeader>
규칙: h1=text-2xl font-bold · 부제=text-sm · 정렬=items-center · 액션 갭=gap-2
제목 앞 아이콘은 전 탭 동일하게 적용 또는 전 탭 미적용 — 현재 Skills·Agents·Sessions만 부분 적용 중(Dashboard로 통일 시 미적용 권장, 제목이 스스로 말하게). 버전/상태 뱃지는 우측 액션 슬롯으로.

2. 간격 리듬 — space-y-6 단일화

현재 space-y-6(12면) vs space-y-4(5면: Dashboard·Agents·Mounts·Projects·ProjectDetail) 분할. 구분 근거 없음 → space-y-6으로 단일화. 카드 그리드 내부는 기존대로 gap-3(KPI)/gap-4(일반) 유지.

3. 버튼 아이콘 간격 — gap 단일화

현재 mr-1(margin) vs flex gap 분할. Button 컴포넌트가 gap를 내장하므로 margin 클래스(mr-1/ml-1) 전면 폐지, 아이콘은 자식으로.

✗ <Plus className="h-4 w-4 mr-1"/> ✓ <Button><Plus className="h-4 w-4"/> 라벨</Button>

4. 사이드바 아이콘 맵 — 중복·의미 정정

두 가지 정정: (a) Skills와 MCP가 같은 Zap 사용 → Skills를 Sparkles 또는 Wand2로. (b) Security = Bell(알림 관례) → ShieldCheck로. 알림은 Header의 MenuClock→NotificationCenter가 담당.

현재
Skills: Zap · MCP: Zap (중복)
Security: Bell
정정
Skills: Sparkles · MCP: Zap
Security: ShieldCheck

5. 탭 패턴 — 무아이콘 + 비래핑 단일화

트리거 아이콘: 다수가 무아이콘이므로 정규 = 무아이콘(MCP만 아이콘 제거). 탭 콘텐츠: 외부 Card로 감싸지 않음(MCP 래핑 제거), 내부 행 카드만. 다중 탭 페이지 간 동일 규칙 적용.

6. 상태(빈/로딩/에러) — 공유 컴포넌트 의무화

EmptyState·ErrorState·loading.tsx 패밀리는 이미 일관적(장점). 규칙을 명시: 인라인 자체 마크업 금지, 색은 success/warning/error/info 시맨틱 토큰 변종만. Dashboard의 if(loading) return early-return도 인라인 조건부로 통일(레이아웃 시프트 감소).

우선순위. 1·2·4가 가장 큰 시각적 분열을 만든다(전 탭 반복). PageHeader 컴포넌트화 하나로 P1·P3·P6 상당부가 사라진다. 사이드바 아이콘 정정(P4)은 한 줄 변경이지만 발견성 향상이 즉각적.
